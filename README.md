## Rúbrica
### Elección del problema
Evidencia consistente. 
Todos los problemas son casos claros para resolverse usando esa técnica y es necesario adatparla, combinarla con otras o extenderla.

### Código
#### Corrección
Evidencia consistente. 
El código resuelve los problemas complejos.

#### Calidad algorítmica y eficiencia
Evidencia consistente. 
Se aplican mejoras en todos los problemas para obtener el código más eficiente posible,
independientemente de que se pueda resolver por fuerza bruta. Enfoques algorítmicos óptimos
o muy eficientes. Uso adecuado de memoización, hashing, grafos, etc.

#### Diseño y claridad del código
Evidencia consistente. 
Código claro, bien estructurado en funciones y clases, con identación correcta, nombres de
variables y métodos bien elegidos, con comentarios que ayudan a seguir el código.

### Memoria
#### Justificación
Evidencia consistente. 
La elección de todos los problemas están correctamente justificada con argumentos claros, técnicos o formativos.

#### Descripción técnica
Evidencia consistente. 
Todos los problemas tienen una descripción técnica que ayudaría a otra persona a entender
cómo se ha resuelto el problema y a seguir todo el proceso de principio a fin.

#### Alternativas
Evidencia consistente. 
En los problemas seleccionados se incluye alguna de las dos cosas: soluciones que se han descartado o alternativas en el planteamiento inicial.

#### Valoración personal
Evidencia consistente. 
El grupo incluye una valoración personal en todos los problemas que da detalles sobre qué les ha aportado la resolución de estos problemas a los miembros del equipo.
